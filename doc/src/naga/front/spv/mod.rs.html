1/*!
2Frontend for [SPIR-V][spv] (Standard Portable Intermediate Representation).
3
4## ID lookups
5
6Our IR links to everything with `Handle`, while SPIR-V uses IDs.
7In order to keep track of the associations, the parser has many lookup tables.
8There map `spv::Word` into a specific IR handle, plus potentially a bit of
9extra info, such as the related SPIR-V type ID.
10TODO: would be nice to find ways that avoid looking up as much
11
12## Inputs/Outputs
13
14We create a private variable for each input/output. The relevant inputs are
15populated at the start of an entry point. The outputs are saved at the end.
16
17The function associated with an entry point is wrapped in another function,
18such that we can handle any `Return` statements without problems.
19
20## Row-major matrices
21
22We don't handle them natively, since the IR only expects column majority.
23Instead, we detect when such matrix is accessed in the `OpAccessChain`,
24and we generate a parallel expression that loads the value, but transposed.
25This value then gets used instead of `OpLoad` result later on.
26
27[spv]: https://www.khronos.org/registry/SPIR-V/
28*/

107impl Instruction {
108    const fn expect(self, count: u16) -> Result<(), Error> {
109        if self.wc == count {
110            Ok(())
111        } else {
112            Err(Error::InvalidOperandCount(self.op, self.wc))

117        self.wc
118            .checked_sub(count)
119            .ok_or(Error::InvalidOperandCount(self.op, self.wc))
120    }

481/// Fragments of Naga IR, to be assembled into `Statements` once data flow is
482/// resolved.
483///
484/// We can't build a Naga `Statement` tree directly from SPIR-V blocks for three
485/// main reasons:
486///
487/// - We parse a function's SPIR-V blocks in the order they appear in the file.
488///   Within a function, SPIR-V requires that a block must precede any blocks it
489///   structurally dominates, but doesn't say much else about the order in which
490///   they must appear. So while we know we'll see control flow header blocks
491///   before their child constructs and merge blocks, those children and the
492///   merge blocks may appear in any order - perhaps even intermingled with
493///   children of other constructs.
494///
495/// - A SPIR-V expression can be used in any SPIR-V block dominated by its
496///   definition, whereas Naga expressions are scoped to the rest of their
497///   subtree. This means that discovering an expression use later in the
498///   function retroactively requires us to have spilled that expression into a
499///   local variable back before we left its scope. (The docs for
500///   [`Frontend::get_expr_handle`] explain this in more detail.)
501///
502/// - We translate SPIR-V OpPhi expressions as Naga local variables in which we
503///   store the appropriate value before jumping to the OpPhi's block.
504///
505/// All these cases require us to go back and amend previously generated Naga IR
506/// based on things we discover later. But modifying old blocks in arbitrary
507/// spots in a `Statement` tree is awkward.
508///
509/// Instead, as we iterate through the function's body, we accumulate
510/// control-flow-free fragments of Naga IR in the [`blocks`] table, while
511/// building a skeleton of the Naga `Statement` tree in [`bodies`]. We note any
512/// spills and temporaries we must introduce in [`phis`].
513///
514/// Finally, once we've processed the entire function, we add temporaries and
515/// spills to the fragmentary `Blocks` as directed by `phis`, and assemble them
516/// into the final Naga `Statement` tree as directed by `bodies`.
517///
518/// [`blocks`]: BlockContext::blocks
519/// [`bodies`]: BlockContext::bodies
520/// [`phis`]: BlockContext::phis
521#[derive(Debug)]
522struct BlockContext<'function> {
523    /// Phi nodes encountered when parsing the function, used to generate spills
524    /// to local variables.
525    phis: Vec<PhiExpression>,
526
527    /// Fragments of control-flow-free Naga IR.
528    ///
529    /// These will be stitched together into a proper [`Statement`] tree according
530    /// to `bodies`, once parsing is complete.
531    ///
532    /// [`Statement`]: crate::Statement
533    blocks: FastHashMap<spirv::Word, crate::Block>,
534
535    /// Map from each SPIR-V block's label id to the index of the [`Body`] in
536    /// [`bodies`] the block should append its contents to.
537    ///
538    /// Since each statement in a Naga [`Block`] dominates the next, we are sure
539    /// to encounter their SPIR-V blocks in order. Thus, by having this table
540    /// map a SPIR-V structured control flow construct's merge block to the same
541    /// body index as its header block, when we encounter the merge block, we
542    /// will simply pick up building the [`Body`] where the header left off.
543    ///
544    /// A function's first block is special: it is the only block we encounter
545    /// without having seen its label mentioned in advance. (It's simply the
546    /// first `OpLabel` after the `OpFunction`.) We thus assume that any block
547    /// missing an entry here must be the first block, which always has body
548    /// index zero.
549    ///
550    /// [`bodies`]: BlockContext::bodies
551    /// [`Block`]: crate::Block
552    body_for_label: FastHashMap<spirv::Word, BodyIndex>,
553
554    /// SPIR-V metadata about merge/continue blocks.
555    mergers: FastHashMap<spirv::Word, MergeBlockInformation>,
556
557    /// A table of `Body` values, each representing a block in the final IR.
558    ///
559    /// The first element is always the function's top-level block.
560    bodies: Vec<Body>,
561
562    /// The module we're building.
563    module: &'function mut crate::Module,
564
565    /// Id of the function currently being processed
566    function_id: spirv::Word,
567    /// Expression arena of the function currently being processed
568    expressions: &'function mut Arena<crate::Expression>,
569    /// Local variables arena of the function currently being processed
570    local_arena: &'function mut Arena<crate::LocalVariable>,
571    /// Arguments of the function currently being processed
572    arguments: &'function [crate::FunctionArgument],
573    /// Metadata about the usage of function parameters as sampling objects
574    parameter_sampling: &'function mut [image::SamplingFlags],
575}

699    fn next_inst(&mut self) -> Result<Instruction, Error> {
700        let word = self.next()?;
701        let (wc, opcode) = ((word >> 16) as u16, (word & 0xffff) as u16);
702        if wc == 0 {
703            return Err(Error::InvalidWordCount);
704        }
705        let op = spirv::Op::from_u32(opcode as u32).ok_or(Error::UnknownInstruction(opcode))?;
706
707        Ok(Instruction { op, wc })
708    }

710    fn next_string(&mut self, mut count: u16) -> Result<(String, u16), Error> {
711        self.temp_bytes.clear();
712        loop {
713            if count == 0 {
714                return Err(Error::BadString);
715            }
716            count -= 1;
717            let chars = self.next()?.to_le_bytes();
718            let pos = chars.iter().position(|&c| c == 0).unwrap_or(4);
719            self.temp_bytes.extend_from_slice(&chars[..pos]);
720            if pos < 4 {
721                break;
722            }
723        }
724        core::str::from_utf8(&self.temp_bytes)
725            .map(|s| (s.to_owned(), count))
726            .map_err(|_| Error::BadString)
727    }

810    /// Return the Naga [`Expression`] to use in `body_idx` to refer to the SPIR-V result `id`.
811    ///
812    /// Ideally, we would just have a map from each SPIR-V instruction id to the
813    /// [`Handle`] for the Naga [`Expression`] we generated for it.
814    /// Unfortunately, SPIR-V and Naga IR are different enough that such a
815    /// straightforward relationship isn't possible.
816    ///
817    /// In SPIR-V, an instruction's result id can be used by any instruction
818    /// dominated by that instruction. In Naga, an [`Expression`] is only in
819    /// scope for the remainder of its [`Block`]. In pseudocode:
820    ///
821    /// ```ignore
822    ///     loop {
823    ///         a = f();
824    ///         g(a);
825    ///         break;
826    ///     }
827    ///     h(a);
828    /// ```
829    ///
830    /// Suppose the calls to `f`, `g`, and `h` are SPIR-V instructions. In
831    /// SPIR-V, both the `g` and `h` instructions are allowed to refer to `a`,
832    /// because the loop body, including `f`, dominates both of them.
833    ///
834    /// But if `a` is a Naga [`Expression`], its scope ends at the end of the
835    /// block it's evaluated in: the loop body. Thus, while the [`Expression`]
836    /// we generate for `g` can refer to `a`, the one we generate for `h`
837    /// cannot.
838    ///
839    /// Instead, the SPIR-V front end must generate Naga IR like this:
840    ///
841    /// ```ignore
842    ///     var temp; // INTRODUCED
843    ///     loop {
844    ///         a = f();
845    ///         g(a);
846    ///         temp = a; // INTRODUCED
847    ///     }
848    ///     h(temp); // ADJUSTED
849    /// ```
850    ///
851    /// In other words, where `a` is in scope, [`Expression`]s can refer to it
852    /// directly; but once it is out of scope, we need to spill it to a
853    /// temporary and refer to that instead.
854    ///
855    /// Given a SPIR-V expression `id` and the index `body_idx` of the [body]
856    /// that wants to refer to it:
857    ///
858    /// - If the Naga [`Expression`] we generated for `id` is in scope in
859    ///   `body_idx`, then we simply return its `Handle<Expression>`.
860    ///
861    /// - Otherwise, introduce a new [`LocalVariable`], and add an entry to
862    ///   [`BlockContext::phis`] to arrange for `id`'s value to be spilled to
863    ///   it. Then emit a fresh [`Load`] of that temporary variable for use in
864    ///   `body_idx`'s block, and return its `Handle`.
865    ///
866    /// The SPIR-V domination rule ensures that the introduced [`LocalVariable`]
867    /// will always have been initialized before it is used.
868    ///
869    /// `lookup` must be the [`LookupExpression`] for `id`.
870    ///
871    /// `body_idx` argument must be the index of the [`Body`] that hopes to use
872    /// `id`'s [`Expression`].
873    ///
874    /// [`Expression`]: crate::Expression
875    /// [`Handle`]: crate::Handle
876    /// [`Block`]: crate::Block
877    /// [body]: BlockContext::bodies
878    /// [`LocalVariable`]: crate::LocalVariable
879    /// [`Load`]: crate::Expression::Load
880    fn get_expr_handle(
881        &self,
882        id: spirv::Word,
883        lookup: &LookupExpression,
884        ctx: &mut BlockContext,
885        emitter: &mut crate::proc::Emitter,
886        block: &mut crate::Block,
887        body_idx: BodyIndex,
888    ) -> Handle<crate::Expression> {
889        // What `Body` was `id` defined in?
890        let expr_body_idx = ctx
891            .body_for_label
892            .get(&lookup.block_id)
893            .copied()
894            .unwrap_or(0);
895
896        // Don't need to do a load/store if the expression is in the main body
897        // or if the expression is in the same body as where the query was
898        // requested. The body_idx might actually not be the final one if a loop
899        // or conditional occurs but in those cases we know that the new body
900        // will be a subscope of the body that was passed so we can still reuse
901        // the handle and not issue a load/store.
902        if is_parent(body_idx, expr_body_idx, ctx) {
903            lookup.handle
904        } else {
905            // Add a temporary variable of the same type which will be used to
906            // store the original expression and used in the current block
907            let ty = self.lookup_type[&lookup.type_id].handle;
908            let local = ctx.local_arena.append(
909                crate::LocalVariable {
910                    name: None,
911                    ty,
912                    init: None,
913                },
914                crate::Span::default(),
915            );
916
917            block.extend(emitter.finish(ctx.expressions));
918            let pointer = ctx.expressions.append(
919                crate::Expression::LocalVariable(local),
920                crate::Span::default(),
921            );
922            emitter.start(ctx.expressions);
923            let expr = ctx
924                .expressions
925                .append(crate::Expression::Load { pointer }, crate::Span::default());
926
927            // Add a slightly odd entry to the phi table, so that while `id`'s
928            // `Expression` is still in scope, the usual phi processing will
929            // spill its value to `local`, where we can find it later.
930            //
931            // This pretends that the block in which `id` is defined is the
932            // predecessor of some other block with a phi in it that cites id as
933            // one of its sources, and uses `local` as its variable. There is no
934            // such phi, but nobody needs to know that.
935            ctx.phis.push(PhiExpression {
936                local,
937                expressions: vec![(id, lookup.block_id)],
938            });
939
940            expr
941        }
942    }

1287    #[allow(clippy::too_many_arguments)]
1288    fn insert_composite(
1289        &self,
1290        root_expr: Handle<crate::Expression>,
1291        root_type_id: spirv::Word,
1292        object_expr: Handle<crate::Expression>,
1293        selections: &[spirv::Word],
1294        type_arena: &UniqueArena<crate::Type>,
1295        expressions: &mut Arena<crate::Expression>,
1296        span: crate::Span,
1297    ) -> Result<Handle<crate::Expression>, Error> {
1298        let selection = match selections.first() {
1299            Some(&index) => index,
1300            None => return Ok(object_expr),
1301        };
1302        let root_span = expressions.get_span(root_expr);
1303        let root_lookup = self.lookup_type.lookup(root_type_id)?;
1304
1305        let (count, child_type_id) = match type_arena[root_lookup.handle].inner {
1306            crate::TypeInner::Struct { ref members, .. } => {
1307                let child_member = self
1308                    .lookup_member
1309                    .get(&(root_lookup.handle, selection))
1310                    .ok_or(Error::InvalidAccessType(root_type_id))?;
1311                (members.len(), child_member.type_id)
1312            }
1313            crate::TypeInner::Array { size, .. } => {
1314                let size = match size {
1315                    crate::ArraySize::Constant(size) => size.get(),
1316                    crate::ArraySize::Pending(_) => {
1317                        unreachable!();
1318                    }
1319                    // A runtime sized array is not a composite type
1320                    crate::ArraySize::Dynamic => {
1321                        return Err(Error::InvalidAccessType(root_type_id))
1322                    }
1323                };
1324
1325                let child_type_id = root_lookup
1326                    .base_id
1327                    .ok_or(Error::InvalidAccessType(root_type_id))?;
1328
1329                (size as usize, child_type_id)
1330            }
1331            crate::TypeInner::Vector { size, .. }
1332            | crate::TypeInner::Matrix { columns: size, .. } => {
1333                let child_type_id = root_lookup
1334                    .base_id
1335                    .ok_or(Error::InvalidAccessType(root_type_id))?;
1336                (size as usize, child_type_id)
1337            }
1338            _ => return Err(Error::InvalidAccessType(root_type_id)),
1339        };
1340
1341        let mut components = Vec::with_capacity(count);
1342        for index in 0..count as u32 {
1343            let expr = expressions.append(
1344                crate::Expression::AccessIndex {
1345                    base: root_expr,
1346                    index,
1347                },
1348                if index == selection { span } else { root_span },
1349            );
1350            components.push(expr);
1351        }
1352        components[selection as usize] = self.insert_composite(
1353            components[selection as usize],
1354            child_type_id,
1355            object_expr,
1356            &selections[1..],
1357            type_arena,
1358            expressions,
1359            span,
1360        )?;
1361
1362        Ok(expressions.append(
1363            crate::Expression::Compose {
1364                ty: root_lookup.handle,
1365                components,
1366            },
1367            span,
1368        ))
1369    }

1371    /// Return the Naga [`Expression`] for `pointer_id`, and its referent [`Type`].
1372    ///
1373    /// Return a [`Handle`] for a Naga [`Expression`] that holds the value of
1374    /// the SPIR-V instruction `pointer_id`, along with the [`Type`] to which it
1375    /// is a pointer.
1376    ///
1377    /// This may entail spilling `pointer_id`'s value to a temporary:
1378    /// see [`get_expr_handle`]'s documentation.
1379    ///
1380    /// [`Expression`]: crate::Expression
1381    /// [`Type`]: crate::Type
1382    /// [`Handle`]: crate::Handle
1383    /// [`get_expr_handle`]: Frontend::get_expr_handle
1384    fn get_exp_and_base_ty_handles(
1385        &self,
1386        pointer_id: spirv::Word,
1387        ctx: &mut BlockContext,
1388        emitter: &mut crate::proc::Emitter,
1389        block: &mut crate::Block,
1390        body_idx: usize,
1391    ) -> Result<(Handle<crate::Expression>, Handle<crate::Type>), Error> {
1392        log::trace!("\t\t\tlooking up pointer expr {pointer_id:?}");
1393        let p_lexp_handle;
1394        let p_lexp_ty_id;
1395        {
1396            let lexp = self.lookup_expression.lookup(pointer_id)?;
1397            p_lexp_handle = self.get_expr_handle(pointer_id, lexp, ctx, emitter, block, body_idx);
1398            p_lexp_ty_id = lexp.type_id;
1399        };
1400
1401        log::trace!("\t\t\tlooking up pointer type {pointer_id:?}");
1402        let p_ty = self.lookup_type.lookup(p_lexp_ty_id)?;
1403        let p_ty_base_id = p_ty.base_id.ok_or(Error::InvalidAccessType(p_lexp_ty_id))?;
1404
1405        log::trace!("\t\t\tlooking up pointer base type {p_ty_base_id:?} of {p_ty:?}");
1406        let p_base_ty = self.lookup_type.lookup(p_ty_base_id)?;
1407
1408        Ok((p_lexp_handle, p_base_ty.handle))
1409    }

1411    #[allow(clippy::too_many_arguments)]
1412    fn parse_atomic_expr_with_value(
1413        &mut self,
1414        inst: Instruction,
1415        emitter: &mut crate::proc::Emitter,
1416        ctx: &mut BlockContext,
1417        block: &mut crate::Block,
1418        block_id: spirv::Word,
1419        body_idx: usize,
1420        atomic_function: crate::AtomicFunction,
1421    ) -> Result<(), Error> {
1422        inst.expect(7)?;
1423        let start = self.data_offset;
1424        let result_type_id = self.next()?;
1425        let result_id = self.next()?;
1426        let pointer_id = self.next()?;
1427        let _scope_id = self.next()?;
1428        let _memory_semantics_id = self.next()?;
1429        let value_id = self.next()?;
1430        let span = self.span_from_with_op(start);
1431
1432        let (p_lexp_handle, p_base_ty_handle) =
1433            self.get_exp_and_base_ty_handles(pointer_id, ctx, emitter, block, body_idx)?;
1434
1435        log::trace!("\t\t\tlooking up value expr {value_id:?}");
1436        let v_lexp_handle = self.lookup_expression.lookup(value_id)?.handle;
1437
1438        block.extend(emitter.finish(ctx.expressions));
1439        // Create an expression for our result
1440        let r_lexp_handle = {
1441            let expr = crate::Expression::AtomicResult {
1442                ty: p_base_ty_handle,
1443                comparison: false,
1444            };
1445            let handle = ctx.expressions.append(expr, span);
1446            self.lookup_expression.insert(
1447                result_id,
1448                LookupExpression {
1449                    handle,
1450                    type_id: result_type_id,
1451                    block_id,
1452                },
1453            );
1454            handle
1455        };
1456        emitter.start(ctx.expressions);
1457
1458        // Create a statement for the op itself
1459        let stmt = crate::Statement::Atomic {
1460            pointer: p_lexp_handle,
1461            fun: atomic_function,
1462            value: v_lexp_handle,
1463            result: Some(r_lexp_handle),
1464        };
1465        block.push(stmt, span);
1466
1467        // Store any associated global variables so we can upgrade their types later
1468        self.record_atomic_access(ctx, p_lexp_handle)?;
1469
1470        Ok(())
1471    }

1473    /// Add the next SPIR-V block's contents to `block_ctx`.
1474    ///
1475    /// Except for the function's entry block, `block_id` should be the label of
1476    /// a block we've seen mentioned before, with an entry in
1477    /// `block_ctx.body_for_label` to tell us which `Body` it contributes to.
1478    fn next_block(&mut self, block_id: spirv::Word, ctx: &mut BlockContext) -> Result<(), Error> {
1479        // Extend `body` with the correct form for a branch to `target`.
1480        fn merger(body: &mut Body, target: &MergeBlockInformation) {
1481            body.data.push(match *target {
1482                MergeBlockInformation::LoopContinue => BodyFragment::Continue,
1483                MergeBlockInformation::LoopMerge | MergeBlockInformation::SwitchMerge => {
1484                    BodyFragment::Break
1485                }
1486
1487                // Finishing a selection merge means just falling off the end of
1488                // the `accept` or `reject` block of the `If` statement.
1489                MergeBlockInformation::SelectionMerge => return,
1490            })
1491        }
1492
1493        let mut emitter = crate::proc::Emitter::default();
1494        emitter.start(ctx.expressions);
1495
1496        // Find the `Body` to which this block contributes.
1497        //
1498        // If this is some SPIR-V structured control flow construct's merge
1499        // block, then `body_idx` will refer to the same `Body` as the header,
1500        // so that we simply pick up accumulating the `Body` where the header
1501        // left off. Each of the statements in a block dominates the next, so
1502        // we're sure to encounter their SPIR-V blocks in order, ensuring that
1503        // the `Body` will be assembled in the proper order.
1504        //
1505        // Note that, unlike every other kind of SPIR-V block, we don't know the
1506        // function's first block's label in advance. Thus, we assume that if
1507        // this block has no entry in `ctx.body_for_label`, it must be the
1508        // function's first block. This always has body index zero.
1509        let mut body_idx = *ctx.body_for_label.entry(block_id).or_default();
1510
1511        // The Naga IR block this call builds. This will end up as
1512        // `ctx.blocks[&block_id]`, and `ctx.bodies[body_idx]` will refer to it
1513        // via a `BodyFragment::BlockId`.
1514        let mut block = crate::Block::new();
1515
1516        // Stores the merge block as defined by a `OpSelectionMerge` otherwise is `None`
1517        //
1518        // This is used in `OpSwitch` to promote the `MergeBlockInformation` from
1519        // `SelectionMerge` to `SwitchMerge` to allow `Break`s this isn't desirable for
1520        // `LoopMerge`s because otherwise `Continue`s wouldn't be allowed
1521        let mut selection_merge_block = None;
1522
1523        macro_rules! get_expr_handle {
1524            ($id:expr, $lexp:expr) => {
1525                self.get_expr_handle($id, $lexp, ctx, &mut emitter, &mut block, body_idx)
1526            };
1527        }
1528        macro_rules! parse_expr_op {
1529            ($op:expr, BINARY) => {
1530                self.parse_expr_binary_op(ctx, &mut emitter, &mut block, block_id, body_idx, $op)
1531            };
1532
1533            ($op:expr, SHIFT) => {
1534                self.parse_expr_shift_op(ctx, &mut emitter, &mut block, block_id, body_idx, $op)
1535            };
1536            ($op:expr, UNARY) => {
1537                self.parse_expr_unary_op(ctx, &mut emitter, &mut block, block_id, body_idx, $op)
1538            };
1539            ($axis:expr, $ctrl:expr, DERIVATIVE) => {
1540                self.parse_expr_derivative(
1541                    ctx,
1542                    &mut emitter,
1543                    &mut block,
1544                    block_id,
1545                    body_idx,
1546                    ($axis, $ctrl),
1547                )
1548            };
1549        }
1550
1551        let terminator = loop {
1552            use spirv::Op;
1553            let start = self.data_offset;
1554            let inst = self.next_inst()?;
1555            let span = crate::Span::from(start..(start + 4 * (inst.wc as usize)));
1556            log::debug!("\t\t{:?} [{}]", inst.op, inst.wc);
1557
1558            match inst.op {
1559                Op::Line => {
1560                    inst.expect(4)?;
1561                    let _file_id = self.next()?;
1562                    let _row_id = self.next()?;
1563                    let _col_id = self.next()?;
1564                }
1565                Op::NoLine => inst.expect(1)?,
1566                Op::Undef => {
1567                    inst.expect(3)?;
1568                    let type_id = self.next()?;
1569                    let id = self.next()?;
1570                    let type_lookup = self.lookup_type.lookup(type_id)?;
1571                    let ty = type_lookup.handle;
1572
1573                    self.lookup_expression.insert(
1574                        id,
1575                        LookupExpression {
1576                            handle: ctx
1577                                .expressions
1578                                .append(crate::Expression::ZeroValue(ty), span),
1579                            type_id,
1580                            block_id,
1581                        },
1582                    );
1583                }
1584                Op::Variable => {
1585                    inst.expect_at_least(4)?;
1586                    block.extend(emitter.finish(ctx.expressions));
1587
1588                    let result_type_id = self.next()?;
1589                    let result_id = self.next()?;
1590                    let _storage_class = self.next()?;
1591                    let init = if inst.wc > 4 {
1592                        inst.expect(5)?;
1593                        let init_id = self.next()?;
1594                        let lconst = self.lookup_constant.lookup(init_id)?;
1595                        Some(ctx.expressions.append(lconst.inner.to_expr(), span))
1596                    } else {
1597                        None
1598                    };
1599
1600                    let name = self
1601                        .future_decor
1602                        .remove(&result_id)
1603                        .and_then(|decor| decor.name);
1604                    if let Some(ref name) = name {
1605                        log::debug!("\t\t\tid={result_id} name={name}");
1606                    }
1607                    let lookup_ty = self.lookup_type.lookup(result_type_id)?;
1608                    let var_handle = ctx.local_arena.append(
1609                        crate::LocalVariable {
1610                            name,
1611                            ty: match ctx.module.types[lookup_ty.handle].inner {
1612                                crate::TypeInner::Pointer { base, .. } => base,
1613                                _ => lookup_ty.handle,
1614                            },
1615                            init,
1616                        },
1617                        span,
1618                    );
1619
1620                    self.lookup_expression.insert(
1621                        result_id,
1622                        LookupExpression {
1623                            handle: ctx
1624                                .expressions
1625                                .append(crate::Expression::LocalVariable(var_handle), span),
1626                            type_id: result_type_id,
1627                            block_id,
1628                        },
1629                    );
1630                    emitter.start(ctx.expressions);
1631                }
1632                Op::Phi => {
1633                    inst.expect_at_least(3)?;
1634                    block.extend(emitter.finish(ctx.expressions));
1635
1636                    let result_type_id = self.next()?;
1637                    let result_id = self.next()?;
1638
1639                    let name = format!("phi_{result_id}");
1640                    let local = ctx.local_arena.append(
1641                        crate::LocalVariable {
1642                            name: Some(name),
1643                            ty: self.lookup_type.lookup(result_type_id)?.handle,
1644                            init: None,
1645                        },
1646                        self.span_from(start),
1647                    );
1648                    let pointer = ctx
1649                        .expressions
1650                        .append(crate::Expression::LocalVariable(local), span);
1651
1652                    let in_count = (inst.wc - 3) / 2;
1653                    let mut phi = PhiExpression {
1654                        local,
1655                        expressions: Vec::with_capacity(in_count as usize),
1656                    };
1657                    for _ in 0..in_count {
1658                        let expr = self.next()?;
1659                        let block = self.next()?;
1660                        phi.expressions.push((expr, block));
1661                    }
1662
1663                    ctx.phis.push(phi);
1664                    emitter.start(ctx.expressions);
1665
1666                    // Associate the lookup with an actual value, which is emitted
1667                    // into the current block.
1668                    self.lookup_expression.insert(
1669                        result_id,
1670                        LookupExpression {
1671                            handle: ctx
1672                                .expressions
1673                                .append(crate::Expression::Load { pointer }, span),
1674                            type_id: result_type_id,
1675                            block_id,
1676                        },
1677                    );
1678                }
1679                Op::AccessChain | Op::InBoundsAccessChain => {
1680                    struct AccessExpression {
1681                        base_handle: Handle<crate::Expression>,
1682                        type_id: spirv::Word,
1683                        load_override: Option<LookupLoadOverride>,
1684                    }
1685
1686                    inst.expect_at_least(4)?;
1687
1688                    let result_type_id = self.next()?;
1689                    let result_id = self.next()?;
1690                    let base_id = self.next()?;
1691                    log::trace!("\t\t\tlooking up expr {base_id:?}");
1692
1693                    let mut acex = {
1694                        let lexp = self.lookup_expression.lookup(base_id)?;
1695                        let lty = self.lookup_type.lookup(lexp.type_id)?;
1696
1697                        // HACK `OpAccessChain` and `OpInBoundsAccessChain`
1698                        // require for the result type to be a pointer, but if
1699                        // we're given a pointer to an image / sampler, it will
1700                        // be *already* dereferenced, since we do that early
1701                        // during `parse_type_pointer()`.
1702                        //
1703                        // This can happen only through `BindingArray`, since
1704                        // that's the only case where one can obtain a pointer
1705                        // to an image / sampler, and so let's match on that:
1706                        let dereference = match ctx.module.types[lty.handle].inner {
1707                            crate::TypeInner::BindingArray { .. } => false,
1708                            _ => true,
1709                        };
1710
1711                        let type_id = if dereference {
1712                            lty.base_id.ok_or(Error::InvalidAccessType(lexp.type_id))?
1713                        } else {
1714                            lexp.type_id
1715                        };
1716
1717                        AccessExpression {
1718                            base_handle: get_expr_handle!(base_id, lexp),
1719                            type_id,
1720                            load_override: self.lookup_load_override.get(&base_id).cloned(),
1721                        }
1722                    };
1723
1724                    for _ in 4..inst.wc {
1725                        let access_id = self.next()?;
1726                        log::trace!("\t\t\tlooking up index expr {access_id:?}");
1727                        let index_expr = self.lookup_expression.lookup(access_id)?.clone();
1728                        let index_expr_handle = get_expr_handle!(access_id, &index_expr);
1729                        let index_expr_data = &ctx.expressions[index_expr.handle];
1730                        let index_maybe = match *index_expr_data {
1731                            crate::Expression::Constant(const_handle) => Some(
1732                                ctx.gctx()
1733                                    .eval_expr_to_u32(ctx.module.constants[const_handle].init)
1734                                    .map_err(|_| {
1735                                        Error::InvalidAccess(crate::Expression::Constant(
1736                                            const_handle,
1737                                        ))
1738                                    })?,
1739                            ),
1740                            _ => None,
1741                        };
1742
1743                        log::trace!("\t\t\tlooking up type {:?}", acex.type_id);
1744                        let type_lookup = self.lookup_type.lookup(acex.type_id)?;
1745                        let ty = &ctx.module.types[type_lookup.handle];
1746                        acex = match ty.inner {
1747                            // can only index a struct with a constant
1748                            crate::TypeInner::Struct { ref members, .. } => {
1749                                let index = index_maybe
1750                                    .ok_or_else(|| Error::InvalidAccess(index_expr_data.clone()))?;
1751
1752                                let lookup_member = self
1753                                    .lookup_member
1754                                    .get(&(type_lookup.handle, index))
1755                                    .ok_or(Error::InvalidAccessType(acex.type_id))?;
1756                                let base_handle = ctx.expressions.append(
1757                                    crate::Expression::AccessIndex {
1758                                        base: acex.base_handle,
1759                                        index,
1760                                    },
1761                                    span,
1762                                );
1763
1764                                if let Some(crate::Binding::BuiltIn(built_in)) =
1765                                    members[index as usize].binding
1766                                {
1767                                    self.gl_per_vertex_builtin_access.insert(built_in);
1768                                }
1769
1770                                AccessExpression {
1771                                    base_handle,
1772                                    type_id: lookup_member.type_id,
1773                                    load_override: if lookup_member.row_major {
1774                                        debug_assert!(acex.load_override.is_none());
1775                                        let sub_type_lookup =
1776                                            self.lookup_type.lookup(lookup_member.type_id)?;
1777                                        Some(match ctx.module.types[sub_type_lookup.handle].inner {
1778                                            // load it transposed, to match column major expectations
1779                                            crate::TypeInner::Matrix { .. } => {
1780                                                let loaded = ctx.expressions.append(
1781                                                    crate::Expression::Load {
1782                                                        pointer: base_handle,
1783                                                    },
1784                                                    span,
1785                                                );
1786                                                let transposed = ctx.expressions.append(
1787                                                    crate::Expression::Math {
1788                                                        fun: crate::MathFunction::Transpose,
1789                                                        arg: loaded,
1790                                                        arg1: None,
1791                                                        arg2: None,
1792                                                        arg3: None,
1793                                                    },
1794                                                    span,
1795                                                );
1796                                                LookupLoadOverride::Loaded(transposed)
1797                                            }
1798                                            _ => LookupLoadOverride::Pending,
1799                                        })
1800                                    } else {
1801                                        None
1802                                    },
1803                                }
1804                            }
1805                            crate::TypeInner::Matrix { .. } => {
1806                                let load_override = match acex.load_override {
1807                                    // We are indexing inside a row-major matrix
1808                                    Some(LookupLoadOverride::Loaded(load_expr)) => {
1809                                        let index = index_maybe.ok_or_else(|| {
1810                                            Error::InvalidAccess(index_expr_data.clone())
1811                                        })?;
1812                                        let sub_handle = ctx.expressions.append(
1813                                            crate::Expression::AccessIndex {
1814                                                base: load_expr,
1815                                                index,
1816                                            },
1817                                            span,
1818                                        );
1819                                        Some(LookupLoadOverride::Loaded(sub_handle))
1820                                    }
1821                                    _ => None,
1822                                };
1823                                let sub_expr = match index_maybe {
1824                                    Some(index) => crate::Expression::AccessIndex {
1825                                        base: acex.base_handle,
1826                                        index,
1827                                    },
1828                                    None => crate::Expression::Access {
1829                                        base: acex.base_handle,
1830                                        index: index_expr_handle,
1831                                    },
1832                                };
1833                                AccessExpression {
1834                                    base_handle: ctx.expressions.append(sub_expr, span),
1835                                    type_id: type_lookup
1836                                        .base_id
1837                                        .ok_or(Error::InvalidAccessType(acex.type_id))?,
1838                                    load_override,
1839                                }
1840                            }
1841                            // This must be a vector or an array.
1842                            _ => {
1843                                let base_handle = ctx.expressions.append(
1844                                    crate::Expression::Access {
1845                                        base: acex.base_handle,
1846                                        index: index_expr_handle,
1847                                    },
1848                                    span,
1849                                );
1850                                let load_override = match acex.load_override {
1851                                    // If there is a load override in place, then we always end up
1852                                    // with a side-loaded value here.
1853                                    Some(lookup_load_override) => {
1854                                        let sub_expr = match lookup_load_override {
1855                                            // We must be indexing into the array of row-major matrices.
1856                                            // Let's load the result of indexing and transpose it.
1857                                            LookupLoadOverride::Pending => {
1858                                                let loaded = ctx.expressions.append(
1859                                                    crate::Expression::Load {
1860                                                        pointer: base_handle,
1861                                                    },
1862                                                    span,
1863                                                );
1864                                                ctx.expressions.append(
1865                                                    crate::Expression::Math {
1866                                                        fun: crate::MathFunction::Transpose,
1867                                                        arg: loaded,
1868                                                        arg1: None,
1869                                                        arg2: None,
1870                                                        arg3: None,
1871                                                    },
1872                                                    span,
1873                                                )
1874                                            }
1875                                            // We are indexing inside a row-major matrix.
1876                                            LookupLoadOverride::Loaded(load_expr) => {
1877                                                ctx.expressions.append(
1878                                                    crate::Expression::Access {
1879                                                        base: load_expr,
1880                                                        index: index_expr_handle,
1881                                                    },
1882                                                    span,
1883                                                )
1884                                            }
1885                                        };
1886                                        Some(LookupLoadOverride::Loaded(sub_expr))
1887                                    }
1888                                    None => None,
1889                                };
1890                                AccessExpression {
1891                                    base_handle,
1892                                    type_id: type_lookup
1893                                        .base_id
1894                                        .ok_or(Error::InvalidAccessType(acex.type_id))?,
1895                                    load_override,
1896                                }
1897                            }
1898                        };
1899                    }
1900
1901                    if let Some(load_expr) = acex.load_override {
1902                        self.lookup_load_override.insert(result_id, load_expr);
1903                    }
1904                    let lookup_expression = LookupExpression {
1905                        handle: acex.base_handle,
1906                        type_id: result_type_id,
1907                        block_id,
1908                    };
1909                    self.lookup_expression.insert(result_id, lookup_expression);
1910                }
1911                Op::VectorExtractDynamic => {
1912                    inst.expect(5)?;
1913
1914                    let result_type_id = self.next()?;
1915                    let id = self.next()?;
1916                    let composite_id = self.next()?;
1917                    let index_id = self.next()?;
1918
1919                    let root_lexp = self.lookup_expression.lookup(composite_id)?;
1920                    let root_handle = get_expr_handle!(composite_id, root_lexp);
1921                    let root_type_lookup = self.lookup_type.lookup(root_lexp.type_id)?;
1922                    let index_lexp = self.lookup_expression.lookup(index_id)?;
1923                    let index_handle = get_expr_handle!(index_id, index_lexp);

4574    /// Walk the statement tree and patch it in the following cases:
4575    /// 1. Function call targets are replaced by `deferred_function_calls` map
4576    fn patch_statements(
4577        &mut self,
4578        statements: &mut crate::Block,
4579        expressions: &mut Arena<crate::Expression>,
4580        fun_parameter_sampling: &mut [image::SamplingFlags],
4581    ) -> Result<(), Error> {
4582        use crate::Statement as S;
4583        let mut i = 0usize;
4584        while i < statements.len() {
4585            match statements[i] {
4586                S::Emit(_) => {}
4587                S::Block(ref mut block) => {
4588                    self.patch_statements(block, expressions, fun_parameter_sampling)?;
4589                }
4590                S::If {
4591                    condition: _,
4592                    ref mut accept,
4593                    ref mut reject,
4594                } => {
4595                    self.patch_statements(reject, expressions, fun_parameter_sampling)?;
4596                    self.patch_statements(accept, expressions, fun_parameter_sampling)?;
4597                }
4598                S::Switch {
4599                    selector: _,
4600                    ref mut cases,
4601                } => {
4602                    for case in cases.iter_mut() {
4603                        self.patch_statements(&mut case.body, expressions, fun_parameter_sampling)?;
4604                    }
4605                }
4606                S::Loop {
4607                    ref mut body,
4608                    ref mut continuing,
4609                    break_if: _,
4610                } => {
4611                    self.patch_statements(body, expressions, fun_parameter_sampling)?;
4612                    self.patch_statements(continuing, expressions, fun_parameter_sampling)?;
4613                }
4614                S::Break
4615                | S::Continue
4616                | S::Return { .. }
4617                | S::Kill
4618                | S::ControlBarrier(_)
4619                | S::MemoryBarrier(_)
4620                | S::Store { .. }
4621                | S::ImageStore { .. }
4622                | S::Atomic { .. }
4623                | S::ImageAtomic { .. }
4624                | S::RayQuery { .. }
4625                | S::SubgroupBallot { .. }
4626                | S::SubgroupCollectiveOperation { .. }
4627                | S::SubgroupGather { .. } => {}
4628                S::Call {
4629                    function: ref mut callee,
4630                    ref arguments,
4631                    ..
4632                } => {
4633                    let fun_id = self.deferred_function_calls[callee.index()];
4634                    let fun_lookup = self.lookup_function.lookup(fun_id)?;
4635                    *callee = fun_lookup.handle;
4636
4637                    // Patch sampling flags
4638                    for (arg_index, arg) in arguments.iter().enumerate() {
4639                        let flags = match fun_lookup.parameters_sampling.get(arg_index) {
4640                            Some(&flags) if !flags.is_empty() => flags,
4641                            _ => continue,
4642                        };
4643
4644                        match expressions[*arg] {
4645                            crate::Expression::GlobalVariable(handle) => {
4646                                if let Some(sampling) = self.handle_sampling.get_mut(&handle) {
4647                                    *sampling |= flags
4648                                }
4649                            }
4650                            crate::Expression::FunctionArgument(i) => {
4651                                fun_parameter_sampling[i as usize] |= flags;
4652                            }
4653                            ref other => return Err(Error::InvalidGlobalVar(other.clone())),
4654                        }
4655                    }
4656                }
4657                S::WorkGroupUniformLoad { .. } => unreachable!(),
4658            }
4659            i += 1;
4660        }
4661        Ok(())
4662    }

5343        &mut self,
5344        inst: Instruction,
5345        module: &mut crate::Module,
5346    ) -> Result<(), Error> {
5347        let start = self.data_offset;
5348        self.switch(ModuleState::Type, inst.op)?;
5349        inst.expect(4)?;
5350        let id = self.next()?;
5351        let type_id = self.next()?;
5352        let length_id = self.next()?;
5353        let length_const = self.lookup_constant.lookup(length_id)?;
5354
5355        let size = resolve_constant(module.to_ctx(), &length_const.inner)
5356            .and_then(NonZeroU32::new)
5357            .ok_or(Error::InvalidArraySize(length_id))?;
5358
5359        let decor = self.future_decor.remove(&id).unwrap_or_default();
5360        let base = self.lookup_type.lookup(type_id)?.handle;
5361
5362        self.layouter.update(module.to_ctx()).unwrap();
5363
5364        // HACK if the underlying type is an image or a sampler, let's assume
5365        //      that we're dealing with a binding-array
5366        //
5367        // Note that it's not a strictly correct assumption, but rather a trade
5368        // off caused by an impedance mismatch between SPIR-V's and Naga's type
5369        // systems - Naga distinguishes between arrays and binding-arrays via
5370        // types (i.e. both kinds of arrays are just different types), while
5371        // SPIR-V distinguishes between them through usage - e.g. given:
5372        //
5373        // ```
5374        // %image = OpTypeImage %float 2D 2 0 0 2 Rgba16f
5375        // %uint_256 = OpConstant %uint 256
5376        // %image_array = OpTypeArray %image %uint_256
5377        // ```
5378        //
5379        // ```
5380        // %image = OpTypeImage %float 2D 2 0 0 2 Rgba16f
5381        // %uint_256 = OpConstant %uint 256
5382        // %image_array = OpTypeArray %image %uint_256
5383        // %image_array_ptr = OpTypePointer UniformConstant %image_array
5384        // ```
5385        //
5386        // ... in the first case, `%image_array` should technically correspond
5387        // to `TypeInner::Array`, while in the second case it should say
5388        // `TypeInner::BindingArray` (kinda, depending on whether `%image_array`
5389        // is ever used as a freestanding type or rather always through the
5390        // pointer-indirection).
5391        //
5392        // Anyway, at the moment we don't support other kinds of image / sampler
5393        // arrays than those binding-based, so this assumption is pretty safe
5394        // for now.
5395        let inner = if let crate::TypeInner::Image { .. } | crate::TypeInner::Sampler { .. } =
5396            module.types[base].inner
5397        {
5398            crate::TypeInner::BindingArray {
5399                base,
5400                size: crate::ArraySize::Constant(size),
5401            }
5402        } else {
5403            crate::TypeInner::Array {
5404                base,
5405                size: crate::ArraySize::Constant(size),
5406                stride: match decor.array_stride {
5407                    Some(stride) => stride.get(),
5408                    None => self.layouter[base].to_stride(),
5409                },
5410            }
5411        };
5412
5413        self.lookup_type.insert(
5414            id,
5415            LookupType {
5416                handle: module.types.insert(
5417                    crate::Type {
5418                        name: decor.name,
5419                        inner,
5420                    },
5421                    self.span_from_with_op(start),
5422                ),
5423                base_id: Some(type_id),
5424            },
5425        );
5426        Ok(())
5427    }

5480    fn parse_type_struct(
5481        &mut self,
5482        inst: Instruction,
5483        module: &mut crate::Module,
5484    ) -> Result<(), Error> {
5485        let start = self.data_offset;
5486        self.switch(ModuleState::Type, inst.op)?;
5487        inst.expect_at_least(2)?;
5488        let id = self.next()?;
5489        let parent_decor = self.future_decor.remove(&id);
5490        let is_storage_buffer = parent_decor
5491            .as_ref()
5492            .is_some_and(|decor| decor.storage_buffer);
5493
5494        self.layouter.update(module.to_ctx()).unwrap();
5495
5496        let mut members = Vec::<crate::StructMember>::with_capacity(inst.wc as usize - 2);
5497        let mut member_lookups = Vec::with_capacity(members.capacity());
5498        let mut storage_access = crate::StorageAccess::empty();
5499        let mut span = 0;
5500        let mut alignment = Alignment::ONE;
5501        for i in 0..u32::from(inst.wc) - 2 {
5502            let type_id = self.next()?;
5503            let ty = self.lookup_type.lookup(type_id)?.handle;
5504            let decor = self
5505                .future_member_decor
5506                .remove(&(id, i))
5507                .unwrap_or_default();
5508
5509            storage_access |= decor.flags.to_storage_access();
5510
5511            member_lookups.push(LookupMember {
5512                type_id,
5513                row_major: decor.matrix_major == Some(Majority::Row),
5514            });
5515
5516            let member_alignment = self.layouter[ty].alignment;
5517            span = member_alignment.round_up(span);
5518            alignment = member_alignment.max(alignment);
5519
5520            let binding = decor.io_binding().ok();
5521            if let Some(offset) = decor.offset {
5522                span = offset;
5523            }
5524            let offset = span;
5525
5526            span += self.layouter[ty].size;
5527
5528            let inner = &module.types[ty].inner;
5529            if let crate::TypeInner::Matrix {
5530                columns,
5531                rows,
5532                scalar,
5533            } = *inner
5534            {
5535                if let Some(stride) = decor.matrix_stride {
5536                    let expected_stride = Alignment::from(rows) * scalar.width as u32;
5537                    if stride.get() != expected_stride {
5538                        return Err(Error::UnsupportedMatrixStride {
5539                            stride: stride.get(),
5540                            columns: columns as u8,
5541                            rows: rows as u8,
5542                            width: scalar.width,
5543                        });
5544                    }
5545                }
5546            }
5547
5548            members.push(crate::StructMember {
5549                name: decor.name,
5550                ty,
5551                binding,
5552                offset,
5553            });
5554        }
5555
5556        span = alignment.round_up(span);
5557
5558        let inner = crate::TypeInner::Struct { span, members };
5559
5560        let ty_handle = module.types.insert(
5561            crate::Type {
5562                name: parent_decor.and_then(|dec| dec.name),
5563                inner,
5564            },
5565            self.span_from_with_op(start),
5566        );
5567
5568        if is_storage_buffer {
5569            self.lookup_storage_buffer_types
5570                .insert(ty_handle, storage_access);
5571        }

5577            id,
5578            LookupType {
5579                handle: ty_handle,
5580                base_id: None,

5582        );

5727                    8 => {
5728                        inst.expect(5)?;
5729                        let high = self.next()?;
5730                        crate::Literal::U64((u64::from(high) << 32) | u64::from(low))
5731                    }
5732                    _ => return Err(Error::InvalidTypeWidth(width as u32)),
5733                }
5734            }
5735            crate::TypeInner::Scalar(crate::Scalar {
5736                kind: crate::ScalarKind::Sint,
5737                width,
5738            }) => {
5739                let low = self.next()?;
5740                match width {
5741                    4 => crate::Literal::I32(low as i32),
5742                    8 => {
5743                        inst.expect(5)?;
5744                        let high = self.next()?;
5745                        crate::Literal::I64(((u64::from(high) << 32) | u64::from(low)) as i64)
5746                    }
5747                    _ => return Err(Error::InvalidTypeWidth(width as u32)),
5748                }
5749            }
5750            crate::TypeInner::Scalar(crate::Scalar {
5751                kind: crate::ScalarKind::Float,
5752                width,
5753            }) => {
5754                let low = self.next()?;
5755                match width {
5756                    // https://registry.khronos.org/SPIR-V/specs/unified1/SPIRV.html#Literal
5757                    // If a numeric type’s bit width is less than 32-bits, the value appears in the low-order bits of the word.
5758                    2 => crate::Literal::F16(f16::from_bits(low as u16)),
5759                    4 => crate::Literal::F32(f32::from_bits(low)),
5760                    8 => {
5761                        inst.expect(5)?;
5762                        let high = self.next()?;
5763                        crate::Literal::F64(f64::from_bits(
5764                            (u64::from(high) << 32) | u64::from(low),
5765                        ))
5766                    }
5767                    _ => return Err(Error::InvalidTypeWidth(width as u32)),
5768                }
5769            }
5770            _ => return Err(Error::UnsupportedType(type_lookup.handle)),
5771        };
5772
5773        let span = self.span_from_with_op(start);
5774
5775        let init = module
5776            .global_expressions
5777            .append(crate::Expression::Literal(literal), span);
5778
5779        self.insert_parsed_constant(module, id, type_id, ty, init, span)
5780    }

6105    /// Record an atomic access to some component of a global variable.
6106    ///
6107    /// Given `handle`, an expression referring to a scalar that has had an
6108    /// atomic operation applied to it, descend into the expression, noting
6109    /// which global variable it ultimately refers to, and which struct fields
6110    /// of that global's value it accesses.
6111    ///
6112    /// Return the handle of the type of the expression.
6113    ///
6114    /// If the expression doesn't actually refer to something in a global
6115    /// variable, we can't upgrade its type in a way that Naga validation would
6116    /// pass, so reject the input instead.
6117    fn record_atomic_access(
6118        &mut self,
6119        ctx: &BlockContext,
6120        handle: Handle<crate::Expression>,
6121    ) -> Result<Handle<crate::Type>, Error> {
6122        log::debug!("\t\tlocating global variable in {handle:?}");
6123        match ctx.expressions[handle] {
6124            crate::Expression::Access { base, index } => {
6125                log::debug!("\t\t  access {handle:?} {index:?}");
6126                let ty = self.record_atomic_access(ctx, base)?;
6127                let crate::TypeInner::Array { base, .. } = ctx.module.types[ty].inner else {
6128                    unreachable!("Atomic operations on Access expressions only work for arrays");
6129                };
6130                Ok(base)
6131            }
6132            crate::Expression::AccessIndex { base, index } => {
6133                log::debug!("\t\t  access index {handle:?} {index:?}");
6134                let ty = self.record_atomic_access(ctx, base)?;
6135                match ctx.module.types[ty].inner {
6136                    crate::TypeInner::Struct { ref members, .. } => {
6137                        let index = index as usize;
6138                        self.upgrade_atomics.insert_field(ty, index);
6139                        Ok(members[index].ty)
6140                    }
6141                    crate::TypeInner::Array { base, .. } => {
6142                        Ok(base)
6143                    }
6144                    _ => unreachable!("Atomic operations on AccessIndex expressions only work for structs and arrays"),
6145                }
6146            }
6147            crate::Expression::GlobalVariable(h) => {
6148                log::debug!("\t\t  found {h:?}");
6149                self.upgrade_atomics.insert_global(h);
6150                Ok(ctx.module.global_variables[h].ty)
6151            }
6152            _ => Err(Error::AtomicUpgradeError(
6153                crate::front::atomic_upgrade::Error::GlobalVariableMissing,
6154            )),
6155        }
6156    }
6157}
